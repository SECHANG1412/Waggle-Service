# Admin Operations Guide

이 문서는 Waggle 관리자 화면에서 문의, 콘텐츠 숨김, 감사 로그를 확인하는 운영 흐름을 정리합니다.

## Entry Point

- 관리자 화면: `/manage`
- 관리자 API: `/manage-api`
- 관리자 진입 버튼은 관리자 계정으로 로그인한 경우에만 노출됩니다.
- 직접 URL로 접근하더라도 백엔드 API에서 관리자 권한을 다시 검증합니다.

## Inquiry Management

관리자는 접수된 문의를 확인하고 처리 상태를 변경할 수 있습니다.

1. `/manage/inquiries`에서 문의 목록을 확인합니다.
2. 문의 상세 내용을 확인합니다.
3. 상태를 변경합니다.
   - `pending`
   - `in_progress`
   - `resolved`
4. 상태 변경 시 `reason`을 입력합니다.
5. 상태 변경 결과는 감사 로그에 기록됩니다.

문의 처리 사유는 사용자가 프로필에서 확인할 수 있습니다.

## Topic Moderation

관리자는 부적절한 토픽을 삭제하지 않고 숨김 처리하거나 숨김 해제할 수 있습니다.

1. `/manage/topics`에서 토픽 목록을 확인합니다.
2. 숨김 또는 숨김 해제를 선택합니다.
3. 조치 사유를 입력합니다.
4. 숨김 처리된 토픽은 일반 목록과 상세 화면에 노출되지 않습니다.
5. 숨김/해제 조치는 감사 로그에 기록됩니다.

토픽은 hard delete 대신 soft hide 방식으로 관리합니다.

## Comment Moderation

관리자는 부적절한 댓글을 숨김 처리하거나 숨김 해제할 수 있습니다.

1. `/manage/comments`에서 댓글 목록을 확인합니다.
2. 숨김 또는 숨김 해제를 선택합니다.
3. 조치 사유를 입력합니다.
4. 숨김 처리된 댓글은 일반 사용자 화면에 그대로 노출되지 않습니다.
5. 숨김/해제 조치는 감사 로그에 기록됩니다.

## Audit Log Review

감사 로그는 관리자 조치 이력을 확인하기 위한 화면입니다.

관리자는 `/manage/logs`에서 다음 정보를 확인할 수 있습니다.

- 관리자 ID
- action
- target type
- target ID
- 변경 전 값
- 변경 후 값
- 조치 사유
- 생성 시각

감사 로그는 문의 상태 변경, 토픽 숨김/해제, 댓글 숨김/해제 같은 주요 관리자 조치를 추적하기 위해 사용합니다.

## User-Facing Result

- 사용자는 프로필에서 본인이 작성한 문의 처리 상태와 처리 사유를 확인할 수 있습니다.
- 사용자는 프로필에서 본인이 작성한 숨김 콘텐츠 상태를 확인할 수 있습니다.
- 콘텐츠 숨김 상세 사유는 사용자에게 그대로 공개하지 않고, 일반 안내 문구와 문의 경로를 제공합니다.

## Closeout Checklist

- 관리자 조치에는 사유가 입력되었는지 확인합니다.
- 숨김 콘텐츠가 일반 사용자 화면에 노출되지 않는지 확인합니다.
- 문의 상태 변경 결과가 사용자 프로필에 반영되는지 확인합니다.
- 감사 로그에서 관리자 ID, 대상, 변경 전/후 값, 사유가 기록되었는지 확인합니다.
